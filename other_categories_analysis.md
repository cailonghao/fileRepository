# 其他核心分类分析报告

基于对项目目录结构及核心代码的深度调研，除已分析的“报价”、“门店”和“客户”外，系统还包含以下五大核心业务分类。

## 1. 供应链与库存管理 (Supply Chain & Inventory)
该分类负责物资的流入、流转与存储，是业务运营的底层支撑。
- **采购管理 (`Purchase`)**: 实现了从订单生成到入库 (`InStore`)、退货 (`BackOutStore`) 的闭环。系统支持从销售单直接生成采购单，实现了业务流的自动衔接。
- **库存管理 (`Stock`)**: 分为仓库 (`Store`) 级别和门店 (`ShopStock`) 级别，支持按期间 (`Period`) 统计库存水位，保证了数据的时效性。
- **供应商管理 (`Supplier`)**: 维护上游供应资源。

## 2. 销售与渠道管理 (Sales & Distribution)
该分类是系统的主要价值产出端，涵盖了多级分销体系。
- **分销与代理**: 通过 `Agent` (代理商) 和 `Dist` (分销商) 模块管理外部渠道。支持品牌授权 (`AgentBrand`) 和分销价格控制 (`DistPrice`)。
- **销售订单 (`Sale`)**: 承接“报价单”后的正式交易环节。包含产品明细 (`SaleProduct`)、出库管理 (`OutStore`) 以及与合同挂钩的审核流程。

## 3. 财务管理 (Financial Management)
财务模块负责所有资金流的核算与记录，确保账实相符。
- **应收/应付 (`Receivable/Payable`)**: 记录预期的资金往来，通常与销售/采购单据自动关联。
- **实收/实付 (`Receive/Pay`)**: 记录真实的资金回笼和支出。
- **费用管理 (`Fee`)**: 处理日常杂费报销及核算。

## 4. 产品与价格动态管理 (Product & Pricing)
不仅管理产品档案，更核心的是其复杂的动态调价机制。
- **基础档案 (`Product`)**: 存储 SKU、规格、单位 (`Unit`) 等基本属性。
- **动态调价 (`PriceAdjust`)**: 系统支持“部门价格调整”、“门店价格调整”、“分销价格调整”等。这种多维度的调价机制允许业务在不同区域、不同渠道执行差异化价格策略。

## 5. 运营汇报与行政工具 (Operations & Tools)
负责企业内部的协同与行政管理。
- **工作汇报 (`Daily/Work`)**: 员工每日报告与阶段性工作汇报，支持评论互动 (`Comment`)。
- **考勤打卡 (`Clock`)**: 门店员工的移动端打卡记录。
- **项目设计 (`Design`)**: 针对定制化需求（如装饰/装修）的专项设计项目管理。
- **通知消息 (`Notify`)**: 全局系统通知与业务提醒。

## 总结：分类间的协作关系
Kapok-Main3 的设计精髓在于模块间的联动：
- **纵向**: 从客户线索 (`Customer`) -> 报价 (`Quote`) -> 销售 (`Sale`) -> 财务应收 (`Receivable`) 的主流程。
- **横向**: 销售缺货 -> 自动触发采购 (`Purchase`) -> 入库 -> 调拨到门店 (`ShopStock`)。
- **交叉**: 不同门店 (`Shop`) 根据其 `ShopPrice` 策略，从同一个 `Product` 池中引用不同的价格执行。

这种高度集成的设计使得系统能够支撑从单体门店到复杂分销体系的全方位业务。
